# Дизайн: плоский общий график зарплаты без лишней вложенности

## Цель

Упростить общий график зарплаты в overview-дашборде так, чтобы:

- визуальная структура стала менее вложенной;
- валютные секции больше не рендерились как отдельные карточки;
- точки на линии не перекрывали друг друга при близких значениях;
- итоговый рендер оставался читаемым на desktop и mobile.

Изменение касается именно общего salary overview chart, а не `salary-progress` блока с треками по валютам и опыту.

## Что меняется

### Визуальная структура

Сейчас compact-вариант общего графика строится через дополнительную вложенность:

- `salary-summary-chart`
- `salary-summary-chart-currencies`
- `salary-summary-chart-currency-card`
- `salary-summary-chart-body salary-summary-chart-body-points`

После изменения график остаётся в контейнере `salary-summary-chart salary-summary-chart-compact salary-summary-chart-points`, но валюты рендерятся как плоские секции внутри общего body без карточек `salary-summary-chart-currency-card`.

Новая структура:

- `salary-summary-chart`
- `salary-summary-chart-head`
- `salary-summary-chart-legend` при наличии legend
- `salary-summary-chart-body`
- `salary-summary-chart-currency-section` для каждой валюты
- `salary-summary-chart-currency-head`
- набор `salary-summary-chart-row`

`salary-summary-chart-currencies` и `salary-summary-chart-currency-card` удаляются из этого сценария рендера.

### Раскладка точек

Горизонтальная позиция точки по-прежнему определяется относительным значением зарплаты через `positionPct`.

Дополнительно вводится вертикальная раскладка точек для общего графика:

- точки обрабатываются слева направо;
- если соседние точки находятся слишком близко по X, они разносятся по разным вертикальным уровням;
- число уровней ограничено небольшим фиксированным числом, ожидаемое значение `3`;
- если уровень свободен в рамках минимального горизонтального зазора, точка ставится на него;
- точка получает не только `left`, но и vertical offset через CSS custom property, например `--salary-point-row`.

Это не меняет смысл данных и не искажает relative scale по X. Меняется только предотвращение визуального наложения.

## UI и CSS

### HTML

В compact-рендере общего графика:

- убрать `salary-summary-chart-currencies`;
- убрать `salary-summary-chart-currency-card`;
- рендерить каждую валюту как плоскую секцию с коротким heading внутри общего `salary-summary-chart-body`;
- сохранить существующие `salary-summary-chart-row`, `salary-summary-chart-line`, `salary-summary-chart-point`.

Для точки добавить inline-style или CSS variable для вертикального уровня.

### CSS

Нужны новые или обновлённые стили:

- `salary-summary-chart-body` должен корректно укладывать последовательность валютных секций;
- `salary-summary-chart-currency-section` должен визуально отделять валюты без card-like контейнера;
- `salary-summary-chart-line` должен получить достаточную высоту под несколько уровней точек;
- `salary-summary-chart-line-track` должен оставаться визуально центрированным относительно новой высоты линии;
- `salary-summary-chart-point` должен позиционироваться по `left` и вертикальному offset;
- compact-стили должны учитывать многослойную раскладку точек на desktop и mobile.

Удаляемые или больше не используемые в этом рендере CSS-классы:

- `salary-summary-chart-currencies`
- `salary-summary-chart-currency-card`

Если эти классы больше нигде не используются, их стили удаляются полностью из source и static CSS.

## Алгоритм раскладки точек

Для общего графика добавляется helper раскладки точек, который работает поверх уже посчитанных `positionPct`.

Требования к helper:

- вход: список точек в порядке возрастания `positionPct`;
- выход: тот же список с добавленным номером вертикального уровня;
- порядок точек по X сохраняется;
- минимальный горизонтальный зазор между точками на одном уровне задаётся константой;
- если точка не помещается на предыдущий уровень, она уходит на следующий;
- если доступных уровней не хватает, используется последний уровень как fallback.

Ожидаемый результат:

- близкие точки не накрывают подписи и маркеры друг друга;
- далёкие точки могут оставаться на одном базовом уровне;
- визуальный рисунок остаётся стабильным при повторном рендере.

## Границы задачи

В задачу входят:

- HTML-структура compact salary overview chart;
- helper вертикальной раскладки точек;
- обновление CSS под более плоский layout;
- проверка в DevTools на реальном отчёте.

В задачу не входят:

- изменение источника salary данных;
- изменение расчёта самих salary metrics;
- изменение `salary-progress` треков;
- изменение других overview-карточек;
- новая интерактивность.

## Тестирование

Нужно покрыть:

1. unit-тест на helper раскладки точек: близкие точки получают разные уровни, порядок по X сохраняется;
2. HTML-тест compact chart: больше не рендерятся `salary-summary-chart-currency-card`;
3. HTML-тест compact chart: новые валютные секции рендерятся в плоском body;
4. regression-проверку на то, что значения `left` остаются относительными и не ломают порядок;
5. ручную проверку через DevTools на desktop и mobile-width:
   - нет перекрытия точек;
   - нет наложения подписей;
   - валютные секции читаются без лишних карточек;
   - rail и точки визуально согласованы.

## Риски и выбранные решения

Основной риск — сделать CSS-перестройку слишком агрессивной и сломать уже существующий non-compact salary chart.

Чтобы этого избежать:

- изменения ограничиваются compact salary overview chart;
- helper вертикальной раскладки применяется только к точечному общему графику;
- CSS должен быть завязан на уже существующие compact-классы, а не менять все salary charts глобально.

## Рекомендованный подход

Реализовать плоский compact-рендер общего графика и отдельный helper вертикальной раскладки точек.

Это лучший вариант, потому что он одновременно:

- уменьшает DOM-вложенность;
- снимает проблему перекрытия точек;
- не требует менять сами salary calculations;
- остаётся совместим с текущей структурой summary rows и existing tests.
