# P0 Checklist для MVP автотестирования отчета

Дата: 10.03.2026  
Статус: черновик для реализации на этапах 4-5

## Обязательные проверки P0

1. Данные из БД читаются без ошибок и содержат обязательные поля для расчетов.
2. Агрегации по месяцам/ролям/опыту совпадают с эталонными значениями.
3. Количество `active` и `archived` вакансий считается корректно.
4. Зарплатные метрики `avg/median/mode/min/max` корректны по каждой ключевой группе.
5. Итоговые строки "Всего" корректно суммируют значения по группам.
6. В `report.html` присутствуют обязательные аналитические секции.
7. Генератор создает обязательные файлы: `report.html`, `styles.css`, `report.*.js`, `plotly-2.27.0.min.js`.
8. При отсутствии части данных отчет не падает и рендерит fallback-состояние.
9. Данные формы "Мои вакансии" корректно записываются/обновляются в БД и читаются обратно без потерь.
10. Расчет эффективности откликов совпадает с эталонной формулой и контрольными данными.
11. Расчет зарплат по исходным валютам совпадает с контрольными значениями.
12. Фильтры по навыкам/валюте/стране/ИТ-аккредитации, сортировка и логика "И"/"ИЛИ" работают корректно.
13. Расчеты топ-навыков и трендов рынка совпадают с эталонными результатами.
14. API `/vacancies`, `/vacancies/{id}` и `/api/vacancies/*` проходят обязательные smoke/contract проверки.
15. DB-инварианты выполняются после операций API/UI (целостность и непротиворечивость данных).
16. Для каждого шага всех автотестов (`api/ui/db/data`) формируются вложения, а retention/очистка применяются по утвержденной политике.

## Критерий прохождения MVP P0

1. 100% прохождение P0-проверок.
2. 0 открытых дефектов severity `critical`/`high` по P0-сценариям.
