## 1) Цель этапа

Преобразовать baseline Этапа 1 в исполнимый тест-дизайн:

1. Разбить требования `RQ-*` на наборы тест-кейсов.
2. Зафиксировать трассировку `требование -> тест-кейс -> тип проверки -> артефакт`.
3. Определить тестовые данные, окружения и предусловия.
4. Выделить MVP-набор для первой реализации автотестов.

## 3) Scope этапа

`In scope`:

1. Декомпозиция `RQ-01` ... `RQ-21` на детальные тест-кейсы.
2. Разделение тестов по слоям: `api`, `db`, `ui`, `logic/data`, `integration`.
3. Определение позитивных, негативных и fallback-сценариев.
4. Определение набора тестовых данных и правил их подготовки.
5. Выделение P0 MVP-набора для первой волны реализации.
6. Определение требований к фикстурам, мокам, SQL-подготовке и тестовому HTTP-запуску отчета.

`Out of scope`:

1. Полная реализация автотестов.
2. Подключение CI pipeline.
3. Массовый рефакторинг продуктового кода ради тестируемости.

## 4) Основные артефакты этапа

По итогам Этапа 2 должны быть готовы:

1. Каталог/таблица детальных тест-кейсов с уникальными ID.
2. Трассировочная матрица `RQ -> TC`.
3. Отдельный список `P0 MVP` тест-кейсов первой реализации.
4. Перечень тестовых данных и инвариантов по каждому набору.
5. Решение по структуре каталогов автотестов и naming convention.

## 5) Предлагаемая схема ID

1. Требования: `RQ-XX` (уже зафиксированы).
2. Тест-кейсы: `TC-API-XXX`, `TC-DB-XXX`, `TC-UI-XXX`, `TC-DATA-XXX`, `TC-INT-XXX`.
3. Наборы данных: `DS-XXX`.
4. Фикстуры/окружения: `ENV-XXX`.

## 6) Рабочий план Этапа 2

1. Зафиксировать шаблон тест-кейса.

Для каждого тест-кейса фиксируются:

1. `ID`
2. `Связанные требования`
3. `Название`
4. `Приоритет`
5. `Тип проверки`
6. `Предусловия`
7. `Тестовые данные`
8. `Шаги`
9. `Ожидаемый результат`
10. `Слой проверки`

12. `Нужные вложения/артефакты`

## 9) Критерии выхода

Этап 2 считается завершенным, когда:

1. Все `RQ-*` имеют хотя бы один связанный тест-кейс.
2. Все P0-требования разложены на позитивные и негативные сценарии.
3. Для DB/API/UI-критичных сценариев определены тестовые данные и предусловия.
4. Согласован MVP-набор первой реализации.
5. Подготовлена трассировка, достаточная для начала реализации автотестов.
